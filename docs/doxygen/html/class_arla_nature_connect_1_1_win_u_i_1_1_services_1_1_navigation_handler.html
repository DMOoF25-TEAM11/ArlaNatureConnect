Detailed Description

Concrete frame-based navigation handler used by the WinUI application. This class implements INavigationHandler and provides the actual navigation behavior using a Frame instance.

Why we have it: This concrete implementation decouples navigation consumers (view-models, services) from platform-specific navigation logic while still providing a simple API. By depending on INavigationHandler, callers become testable and DI-friendly.

How to use:

  • Register the implementation in the DI container (for example NavigationHandler).
  • Initialize the implementation once with the application's main Frame (typically from the main window).
  • Inject INavigationHandler into view-models or services and call Navigate(Type, object?), GoBack,. or inspect CanGoBack as needed.

Example:

// Register in Program/Startup
services.AddSingleton<INavigationHandler, NavigationHandler>();
// Initialize (MainWindow code-behind)
var nav = App.HostInstance.Services.GetRequiredService<INavigationHandler>();
nav.Initialize(this.MainFrame);
// Usage in a view-model
public class LoginViewModel
{
private readonly INavigationHandler _navigation;
public LoginViewModel(INavigationHandler navigation) { _navigation = navigation; }
public void OnLoginSuccess() => _navigation.Navigate(typeof(HomePage), userRole);
}
