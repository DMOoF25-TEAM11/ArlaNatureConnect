Detailed Description

Centralizes app-wide status reporting so multiple components can signal loading and database-connection state in a consistent way. This prevents UI flicker and race conditions by keeping IsLoadingOrSaving true until every independent caller finishes (reference-counted tokens), so the UI and other consumers observe reliable state.

How to use:

  • Call BeginLoadingOrSaving when a component starts an operation that should mark the app as loading. Dispose the returned token when the operation completes.
  • Read IsLoadingOrSaving to know whether any active loading callers remain.
  • Set or read HasDbConnection to indicate whether a database connection is available.

Why we have it:

  • To provide a single source of truth for global loading/db connection state so multiple independent components do not fight over UI indicators and can avoid flicker.
  • To provide change notifications via PropertyChanged and StatusInfoChanged so UIs and other services can react to status changes.

Acquire a loading token. When the returned IDisposable is disposed the internal loading count is decremented. Use this when multiple independent callers need to indicate "loading" and you want IsLoadingOrSaving to be true until every caller has finished (disposed their token).

Typical usage:

using (statusService.BeginLoadingOrSaving())
{
await SomeWorkAsync();
}

◆ IsLoadingOrSaving

bool ArlaNatureConnect.Core.Services.StatusInfoService.IsLoadingOrSaving
get

True while one or more callers have an active loading token from BeginLoadingOrSaving. Setting this property directly will replace the internal reference-count with 0/1.

How to use:

  • Prefer BeginLoadingOrSaving to participate in reference-counted loading.
  • Read this property to drive UI indicators (e.g. spinners).

Why we have it:

  • Ensures UI shows loading only while there are active operations, preventing flicker caused by short-lived operations starting and stopping rapidly.
