Detailed Description

Abstraction for frame-based navigation used by the WinUI application.

Why we have it: This interface decouples navigation consumers (view-models, services) from the concrete navigation implementation. It makes the navigation surface testable, swappable and easy to register in the DI container.

How to use:

  • Register a concrete implementation in the DI container (for example NavigationHandler).
  • Initialize the implementation once with the application's main Frame (typically from the main window).
  • Inject INavigationHandler into view-models or services and call Navigate, GoBack, or inspect CanGoBack as needed.

Example (registration, initialization and usage):

// In Program/Startup when configuring services
services.AddSingleton<INavigationHandler, NavigationHandler>();
// In MainWindow code-behind after frame has been created
var nav = App.HostInstance.Services.GetRequiredService<INavigationHandler>();
nav.Initialize(this.MainFrame);
// In a view-model or command
public class LoginViewModel
{
private readonly INavigationHandler _navigation;
public LoginViewModel(INavigationHandler navigation) { _navigation = navigation; }
public void OnLoginSuccess() => _navigation.Navigate(typeof(HomePage), userRole);
}
