Detailed Description

SQL Server repository for Role lookups used across UC flows.

How to use and why we have it:

  • Use this repository to perform lookups and CRUD operations for Role entities.
  • It encapsulates data-access concerns and keeps callers decoupled from EF Core details.
  • When possible use <inheritdoc/> on implementing members to inherit documentation from interfaces or base types so documentation stays consistent across implementations.

Code example:

// create factory (DI normally provides this)
IDbContextFactory<AppDbContext> factory = /* ... *&zwj;/;
var repo = new RoleRepository(factory);
Role? r = await repo.GetByNameAsync("Farmer");
if (r != null) Console.WriteLine(r.Name);

Find a Role by its name.

This method performs a case-sensitive lookup of the role name using a short-lived AppDbContext created from the injected IDbContextFactory<AppDbContext>.
