\doxysubsection{Detailed Description}
Centralizes app-\/wide status reporting so multiple components can signal loading and database-\/connection state in a consistent way. This prevents UI flicker and race conditions by keeping {\ttfamily Is\+Loading\+Or\+Saving} true until every independent caller finishes (reference-\/counted tokens), so the UI and other consumers observe reliable state. 

How to use\+:
\begin{DoxyItemize}
\item Call \doxylink{class_arla_nature_connect_1_1_core_1_1_services_1_1_status_info_service_a8325649a903df53b29fd5cca9081b19b}{Begin\+Loading\+Or\+Saving} when a component starts an operation that should mark the app as loading. Dispose the returned token when the operation completes.
\item Read \doxylink{class_arla_nature_connect_1_1_core_1_1_services_1_1_status_info_service_a14a9927fde51d0b81532f2f331192e45}{Is\+Loading\+Or\+Saving} to know whether any active loading callers remain.
\item Set or read \doxylink{class_arla_nature_connect_1_1_core_1_1_services_1_1_status_info_service_acfa61e3af8ce29dd3eebf13006d055ce}{Has\+Db\+Connection} to indicate whether a database connection is available.
\end{DoxyItemize}

Why we have it\+:
\begin{DoxyItemize}
\item To provide a single source of truth for global loading/db connection state so multiple independent components do not fight over UI indicators and can avoid flicker.
\item To provide change notifications via \doxylink{class_arla_nature_connect_1_1_core_1_1_services_1_1_status_info_service_a25c6d6f7ffef4c2dc3746ee05514ffd3}{Property\+Changed} and \doxylink{class_arla_nature_connect_1_1_core_1_1_services_1_1_status_info_service_a1fbe9997e086f552078af48be24fa9da}{Status\+Info\+Changed} so UIs and other services can react to status changes.
\end{DoxyItemize}
