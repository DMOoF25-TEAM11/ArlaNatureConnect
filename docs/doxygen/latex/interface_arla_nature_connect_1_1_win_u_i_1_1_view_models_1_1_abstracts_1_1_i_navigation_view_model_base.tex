\doxysubsection{Detailed Description}
Contract for view-\/models that drive role-\/aware navigation and side-\/menu integration in \doxylink{namespace_arla_nature_connect_1_1_win_u_i}{Win\+UI} pages. 

Why this interface exists\+:
\begin{DoxyItemize}
\item Provides a minimal surface that pages and controls can depend on without referencing concrete view-\/model types.
\item Encapsulates navigation-\/related commands, observable properties and lifecycle hooks used by the application\textquotesingle{}s navigation pages (for example pages that display dashboards, lists and task content per user/role).
\item Makes it easier to write unit tests and UI components that operate against an interface rather than concrete implementations, enabling mocking and decoupling between views and view-\/model implementations.
\end{DoxyItemize}\hypertarget{interface_arla_nature_connect_1_1_win_u_i_1_1_view_models_1_1_abstracts_1_1_i_navigation_view_model_base_autotoc_md0}{}\doxysubsubsection{\texorpdfstring{Inheriting XML Documentation with }{Inheriting XML Documentation with }}\label{interface_arla_nature_connect_1_1_win_u_i_1_1_view_models_1_1_abstracts_1_1_i_navigation_view_model_base_autotoc_md0}

This keeps documentation DRY (don\textquotesingle{}t repeat yourself) and ensures that the contract and its implementors remain synchronized when remarks change. 

How to use and why we have it\+:
