\doxysubsection{Detailed Description}
Abstraction for frame-\/based navigation used by the \doxylink{namespace_arla_nature_connect_1_1_win_u_i}{Win\+UI} application. 

Why we have it\+: This interface decouples navigation consumers (view-\/models, services) from the concrete navigation implementation. It makes the navigation surface testable, swappable and easy to register in the DI container.

How to use\+:
\begin{DoxyItemize}
\item Register a concrete implementation in the DI container (for example {\ttfamily \doxylink{class_arla_nature_connect_1_1_win_u_i_1_1_services_1_1_navigation_handler}{Navigation\+Handler}}).
\item Initialize the implementation once with the application\textquotesingle{}s main {\ttfamily Frame} (typically from the main window).
\item Inject {\ttfamily \doxylink{interface_arla_nature_connect_1_1_win_u_i_1_1_services_1_1_i_navigation_handler}{INavigation\+Handler}} into view-\/models or services and call \doxylink{interface_arla_nature_connect_1_1_win_u_i_1_1_services_1_1_i_navigation_handler_a5abc4891e86edbf8164afc6555f99c44}{Navigate}, \doxylink{interface_arla_nature_connect_1_1_win_u_i_1_1_services_1_1_i_navigation_handler_aafe1704d1fe61fbf7ac01b54f9926081}{Go\+Back}, or inspect \doxylink{interface_arla_nature_connect_1_1_win_u_i_1_1_services_1_1_i_navigation_handler_a8c496204b528be5bc3d9a9e5b2340498}{Can\+Go\+Back} as needed. 
\end{DoxyItemize}
