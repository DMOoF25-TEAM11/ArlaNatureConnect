\doxysubsection{Detailed Description}
Provides a simple application-\/wide message store that exposes informational and error messages for the UI and other components. 

This interface centralizes how components communicate status and error information across the application. Implementations typically hold transient messages that can be bound to the UI and are observable via INotify\+Property\+Changed so views can react when messages are added, removed or cleared. The service separates informational (status) messages from error messages and optionally associates messages with a specific entity via \doxylink{interface_arla_nature_connect_1_1_core_1_1_services_1_1_i_app_message_service_a79f436975260633ee8e3209a0c04b2fa}{Entity\+Name}.

Reasons for having this interface\+:
\begin{DoxyItemize}
\item Decouples message producers (services, view models) from message consumers (views).
\item Enables easy unit testing by mocking the interface.
\item Provides a single place to manage message lifecycle (add, clear) and to raise property change notifications for UI binding. 
\end{DoxyItemize}
