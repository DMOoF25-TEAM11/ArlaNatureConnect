**Use Case ID:** UC03  
**Name:** Create Nature Check  
**Scope:** System prototype (Arla Nature Connect)  
**Level:** User goal  

**Primary Actors:**  
- Farmer  
- Consultant  
- Arla Employee  

**Description:**  
The user selects a role (Landmand, Konsulent, or Arla Medarbejder) on the front page to access the system.  
No password or email authentication is required, as this is a prototype.  
Selection the Consultant role allows the user to suggest dates for nature checks and create new nature checks. 
Farmers then get notified of the suggested nature checks and can accept or decline them.

**Preconditions:**  
- The system is running and displays the start page.  

**Postconditions:**  
- The nature check date is set and the farmer is notified.
- The farmer can accept or decline the nature check.
