# Brief Use Case – UC002B.5: Delete Farm

**Use Case ID:** UC002B.5  
**Use Case Name:** Delete Farm  
**Scope:** System prototype (Arla Nature Connect)  
**Level:** User goal  
**Primary Actor:** Arla Employee (Admin / Sustainability Staff)

---

## Brief Description

The Arla employee deletes a farm from the system. The system prevents deletion if the farm has active Nature Check Cases.

---

## Preconditions

1. The Arla employee is authenticated in the system (UC01).
2. The farm exists in the system.
3. The farm has no active Nature Check Cases (status "Assigned" or "InProgress").

---

## Postconditions

1. The farm is removed from the system.
2. The farm no longer appears in the farm list.
3. The farm cannot be selected for new assignments.

---

## Main Success Scenario

1. The employee selects a farm to delete.
2. The employee clicks delete.
3. The system checks for active Nature Check Cases.
4. The system confirms deletion.
5. The system deletes the farm.
6. The system updates the farm list.

---

## Extensions

### 3a. Farm has active cases:
The system displays error: "Gården kan ikke slettes fordi den har aktive Natur Check opgaver." → Deletion is prevented.

---

## Technology Notes

- The system uses Entity Framework Core repositories for all database operations (not stored procedures).
- Foreign key constraints and cascade delete are configured in EF Core entity relationships.
