# Casual Use Case – UC002B.6: View Consultant Notifications

**Use case:** UC002B.6 – View Consultant Notifications  
**Scope:** The prototype of the Arla Nature Connect system  
**Level:** User goal  
**Primary Actors:** Consultant

---

### **Stakeholders and Interests**

- **Consultants** want to see all assigned cases to plan their work.

---

### **Preconditions**

- The consultant is authenticated (UC01).  
- The consultant has the Consultant role.

---

### **Postconditions**

- All assigned cases are displayed.  
- Cases are sorted by assignment date.  
- Notifications are marked as read.

---

### **Main Success Scenario**

1. The consultant opens the notifications view.  
2. The system loads assigned cases.  
3. The system displays cases sorted by date.  
4. The system marks notifications as read.

---

### **Extensions**

- **No cases assigned:** System displays empty list.

---

### **Special Requirements**

- Cases sorted by assignment date (newest first).
- Notifications marked as read when viewed.
