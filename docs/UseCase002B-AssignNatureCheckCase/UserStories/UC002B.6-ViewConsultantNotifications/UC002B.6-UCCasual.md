# Casual Use Case – UC002B.6: View Consultant Notifications

**Use case:** UC002B.6 – View Consultant Notifications  
**Scope:** The prototype of the Arla Nature Connect system  
**Level:** User goal  
**Primary Actors:** Consultant

---

### **Stakeholders and Interests**

- **Consultants** want to see all assigned cases to plan their work.

---

### **Preconditions**

- The consultant is authenticated (UC01).  
- The consultant has the Consultant role.

---

### **Postconditions**

- All assigned cases are displayed.  
- Cases are sorted by assignment date (newest first).  
- ConsultantNotificationDto objects are generated from NatureCheckCase data (not stored in database).

---

### **Main Success Scenario**

1. The consultant opens the notifications view.  
2. The system loads assigned cases via EF Core repositories.  
3. The system loads related farm information.
4. The system generates ConsultantNotificationDto objects from case data (not stored in database).
5. The system displays cases sorted by assignment date (newest first).

---

### **Extensions**

- **No cases assigned:** System displays empty list.

---

### **Special Requirements**

- Cases sorted by assignment date (newest first).
- Notifications are generated from NatureCheckCase data when consultant views them (not stored in database).
- No "mark as read" functionality - notifications are always current based on case data.
- All database operations use Entity Framework Core repositories (not stored procedures).
