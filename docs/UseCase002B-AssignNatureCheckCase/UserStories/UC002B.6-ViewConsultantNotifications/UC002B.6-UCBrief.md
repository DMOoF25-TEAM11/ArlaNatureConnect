**Use Case ID:** UC002B.6  
**Use Case Name:** View Consultant Notifications  
**Scope:** System prototype (Arla Nature Connect)  
**Level:** User goal  
**Primary Actor:** Consultant

---

## Brief Description

The consultant views all Nature Check Case assignments that have been assigned to them. Cases are displayed with farm information, assignment date, priority, and notes, sorted by assignment date.

---

## Preconditions

1. The consultant is authenticated in the system (UC01).
2. The consultant has the Consultant role.
3. The consultant has at least one assigned Nature Check Case (or empty list is shown).

---

## Postconditions

1. All Nature Check Cases assigned to the consultant are loaded and displayed.
2. Cases are sorted by assignment date (newest first).
3. ConsultantNotificationDto objects are generated from NatureCheckCase data (not stored in database).
4. The consultant can see farm name, assignment date, priority, and notes for each case.

---

## Main Success Scenario

1. The consultant opens the notifications view.
2. The system loads all Nature Check Cases assigned to the consultant via EF Core repositories.
3. The system loads related farm information.
4. The system generates ConsultantNotificationDto objects from case data (not stored in database).
5. The system sorts cases by assignment date (newest first).
6. The system displays cases with farm information, priority, and notes.

---

## Extensions

### 2a. No cases assigned:
The system displays an empty list with message "Ingen opgaver tildelt."

---

## Technology Notes

- Notifications are generated from NatureCheckCase data when consultant views them (not stored in database).
- ConsultantNotificationDto objects are created in-memory from case data.
- Cases are sorted by AssignedAt date (newest first).
- No "mark as read" functionality - notifications are always current based on case data.
