# UC002B.6 Domain Model

Domain Model for View Consultant Notifications use case. This builds upon UC002B.5.

```mermaid
---
title: Use Case 002B.6 - Domain Model (Builds on UC002B.5)
---
classDiagram
    direction TB

    class Person {
        FirstName
        LastName
        Email
    }

    class NatureCheckCase {
        Status
        Priority
        Notes
        AssignedAt
    }

    class Notification {
        CaseId
        ConsultantId
        CreatedAt
        IsRead
    }

    class Farm {
        Name
        CVR
    }

    %% Relationships
    Person "1" -- "0..*" NatureCheckCase : assignedTo
    NatureCheckCase "1" -- "1" Notification : generates
    Person "1" -- "0..*" Notification : receives
    NatureCheckCase "1" -- "1" Farm : for
```

**New Domain Concepts (UC002B.6):**
- **Notification View** - The view of assigned cases for a consultant
- **Read Status** - Whether a notification has been viewed

**Business Rules:**
- Cases are sorted by AssignedAt date (newest first)
- Notifications are marked as read when viewed
