# Casual Use Case – UC002B.4: Create Farm (Inline)

**Use case:** UC002B.4 – Create Farm (Inline)  
**Scope:** The prototype of the Arla Nature Connect system  
**Level:** Subfunction  
**Primary Actors:** Arla Employee (Admin / Sustainability Staff)

---

### **Stakeholders and Interests**

- **Arla employees** want to create farms inline during assignment to avoid switching contexts.

---

### **Preconditions**

- The Arla employee is authenticated (UC01).  
- The employee is in the assignment process.  
- All required farm information is provided.

---

### **Postconditions**

- A new farm is created in the system.  
- The farm is available for selection in the assignment process.

---

### **Main Success Scenario**

1. The employee clicks "Tilføj ny gård" during assignment.  
2. The system displays the farm creation form.  
3. The employee enters farm details and submits.  
4. The system validates and creates the farm.  
5. The employee returns to assignment with the new farm available.

---

### **Extensions**

- **CVR already exists:** System shows error, employee must change CVR.  
- **Owner email exists but not Farmer role:** System shows error, employee must use different email.

---

### **Special Requirements**

- CVR must be unique.
- A farmer can own multiple farms.
- All database operations use Entity Framework Core repositories (not stored procedures).
