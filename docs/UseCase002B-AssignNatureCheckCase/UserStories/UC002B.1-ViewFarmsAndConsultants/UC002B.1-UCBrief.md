# Brief Use Case – UC002B.1: View Farms and Consultants for Assignment

**Use Case ID:** UC002B.1  
**Use Case Name:** View Farms and Consultants for Assignment  
**Scope:** System prototype (Arla Nature Connect)  
**Level:** User goal  
**Primary Actor:** Arla Employee (Admin / Sustainability Staff)

---

## Brief Description

The Arla employee opens the Nature Check Cases module and views a list of all farms and consultants in the system. Each farm displays its assignment status (whether it has an active Nature Check Case) and, if applicable, the assigned consultant and priority level.

---

## Preconditions

1. The Arla employee is authenticated in the system (UC01).
2. The employee has opened the Nature Check Cases module.
3. The system has access to farm and consultant data.

---

## Postconditions

1. All farms in the system are loaded and displayed to the employee.
2. Each farm shows its assignment status (whether it has an active Nature Check Case or not).
3. Each farm with an active case shows the assigned consultant's name and priority level.
4. All consultants (persons with Consultant role) in the system are loaded and displayed to the employee.
5. Consultants are sorted alphabetically by name.
6. Farms are sorted alphabetically by name.

---

## Main Success Scenario

1. The Arla employee opens the *Nature Check Cases* module.
2. The system loads all farms from the database.
3. The system loads all active Nature Check Cases.
4. The system loads all consultants (persons with Consultant role).
5. The system aggregates farm data with assignment status and consultant information.
6. The system displays the farm list with assignment status indicators.
7. The system displays the consultant list sorted alphabetically.

---

## Extensions

### 2a. No farms exist in system:
The system displays an empty farm list with a message "Ingen gårde fundet. Opret en gård for at fortsætte."

### 2b. No consultants exist in system:
The system displays an empty consultant list with a message "Ingen konsulenter fundet."

### 3a. Database error occurs:
The system displays an error message "Kunne ikke indlæse data. Prøv igen senere."

---

## Technology Notes

- Data aggregation: Service aggregates Farm + Person + Address + Case data into DTOs for efficient UI display.
- Assignment status is calculated by checking for active cases (status "Assigned" or "InProgress").
- Priority badges are displayed for farms with active cases.
